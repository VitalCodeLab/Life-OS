# 数据记录规范 V1.0

## 一、总则

- **目的**：为分析提供可靠原料，而非追求美观。
- **原则**：**最小必要记录**。只记录为验证假设和评估进展所必需的数据。
- **工具**：飞书多维表格（主数据库）、GitHub（归档与版本化）。

## 二、数据表结构（飞书）

### 1. 表：`每日核心数据`

- **定位**：系统级仪表盘，俯瞰所有项目健康状况。
- **字段**：
  | 字段名 | 类型 | 说明 | 必填 |
  | :--- | :--- | :--- | :--- |
  | 日期 | 日期 | 自动生成当天 | 是 |
  | 精力值 (1-5) | 数字 | 晨起主观感受 | 是 |
  | 睡眠时长 (h) | 数字 | | 是 |
  | **模式** | 单选 | `项目主导` / `工作主导` | 是 |
  | 核心问题 | 文本 | 当日最想解决的一个 Life-OS 层面问题 | 否 |
  | 备注 | 文本 | 任何异常情况 | 否 |

### 2. 表：`[项目名]_日志` (如 `康复日志`)

- **定位**：项目级详细数据。
- **字段示例（康复项目）**：
  | 字段名 | 类型 | 说明 |
  | :--- | :--- | :--- |
  | 日期 | 日期 | 链接到`每日核心数据` |
  | 无痛久坐时长 (min) | 数字 | **核心指标** |
  | 疼痛感 (1-10) | 数字 | 睡前评估 |
  | 训练完成度 | 单选 | `完成`/`部分`/`未做` |
  | 训练内容 | 文本 | 简述，如“晨间序列+臀桥 3 组” |
  | 身体感受 | 文本 | 自由记录，如“左侧臀部有牵拉感” |
  | 明日调整 | 文本 | 基于今日感受的微调计划 |

## 三、记录流程

1.  **晨间 (3 分钟)**：打开飞书仪表盘，填写`每日核心数据`中的“精力值”、“模式”，提出“核心问题”。
2.  **晚间 (5 分钟)**：填写当日`项目日志`的核心数据与简要感受。
3.  **每周**：利用飞表视图快速生成数据透视，用于复盘。
