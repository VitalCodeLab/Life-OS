# Life-OS 核心理念与基本协议

## 一、为什么建造这个系统？

（阐述你从“追逐变现”到“构建价值”的转变，以及公开建造、系统化解决问题的初衷）

## 二、三条核心原则

1.  **原则一：价值前置，而非流量前置**

    - 定义：专注于解决一个真实、具体的问题，并将解决方案构建成可复用的产品（文档、协议、数据）。流量是构建过程的自然副产品。
    - 行动：每个分支项目必须始于一个清晰的、可衡量的“问题陈述”。

2.  **原则二：系统驱动，而非意志力驱动**

    - 定义：依赖设计好的协议、模板和自动化流程来降低执行阻力，而不是依靠间歇性爆发的动力。
    - 行动：为所有重复性工作（如记录、复盘）创建标准化模板，并集成到日常工具流中。

3.  **原则三：开源协作，而非孤军奋战**
    - 定义：在项目成熟后，将方法和数据开源，邀请多元视角进行验证、批评与共同迭代，以对抗个人盲区。
    - 行动：项目文档必须清晰、结构化；关键决策和数据分析过程必须公开。

## 三、基本工作协议

- **信息输入管理**：所有灵感、待读、待看，统一进入飞书“闪念收集”文档，每周清空一次。
- **输出节奏**：以“项目”为单位进行输出。每 2-4 周，每个活跃项目至少产出一份“里程碑报告”（视频或图文）。
- **状态标记**：在项目 README 中使用统一的状态徽章：`🟢 活跃` / `🟡 暂停` / `🔴 归档` / `🟣 寻求协作`。
